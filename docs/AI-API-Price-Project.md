# AI API Price 项目文档

## 项目概述

**项目名称**: AI API Price (AI 模型价格对比/查询工具)

**核心概念**: 创建一个专门查询和对比各大 AI 厂商 API 价格的工具站，帮助用户快速了解不同 AI 模型的价格信息。

---

## 关键词分析

### 目标关键词
- **AI API Price** - 主关键词
- **AI API pricing**
- **LLM API cost**
- **AI model price comparison**
- **ChatGPT API price**
- **Claude API price**
- **AI 接口价格** (中文)

### 搜索意图分析
用户搜索这些关键词的核心需求：
1. **快速查询价格** - 想知道某个具体 AI 模型的价格
2. **对比不同模型** - 在多个模型之间做价格比较
3. **成本估算** - 计算自己使用场景下的 API 成本
4. **发现便宜替代品** - 寻找性价比更高的模型

## 竞争对手分析

### 参考网站：https://pricepertoken.com/

#### 网站定位
- 专注于 AI/LLM API 价格对比
- 按 token 计费模式展示
- 覆盖主流 AI 厂商（OpenAI, Anthropic, Google, Meta 等）

#### 核心功能
1. **价格列表** - 按厂商/模型展示价格
2. **价格计算器** - 输入 token 数量计算成本
3. **对比功能** - 多模型价格横向对比
4. **价格历史** - 追踪价格变化趋势

#### 流量估计
- 月访问量：~50,000 - 100,000
- 主要流量来源：Google 自然搜索
- 目标用户：开发者、企业技术决策者

#### 变现方式
1. **Google AdSense** - 展示广告
2. **联盟营销** - AI 厂商推荐链接
3. **付费 API** - 提供价格数据 API
4. **赞助展示** - 厂商付费置顶

#### 可借鉴的设计元素
- 清晰的价格表格
- 实时价格更新
- 直观的对比视图
- 计算器工具增加用户停留时间

## 收入预估

### 流量假设
| 场景 | 月访问量 | 页面浏览量 |
|------|----------|------------|
| 保守 | 10,000 | 30,000 |
| 中等 | 50,000 | 150,000 |
| 乐观 | 100,000 | 300,000 |

### AdSense 收入估算
- **RPM 假设**: $5 - $15 (技术类流量 RPM 较高)
- **保守**: 30,000 PV × $8 RPM / 1000 = **$240/月**
- **中等**: 150,000 PV × $10 RPM / 1000 = **$1,500/月**
- **乐观**: 300,000 PV × $12 RPM / 1000 = **$3,600/月**

### 其他收入来源
| 来源 | 保守 | 中等 | 乐观 |
|------|------|------|------|
| 联盟营销 | $50 | $300 | $800 |
| 赞助展示 | $0 | $500 | $2,000 |
| 付费 API | $0 | $200 | $1,000 |
| **合计** | **$290** | **$2,500** | **$7,400** |

### 成本估算
| 项目 | 月成本 |
|------|--------|
| 域名 | $1 |
| 托管 (Vercel) | $0 - $20 |
| 数据库 (Supabase) | $0 - $25 |
| API 数据抓取 | $0 (自建) |
| **合计** | **$1 - $46** |

### ROI 分析
- **初期投入**: 主要是开发时间
- **运营成本**: 极低（静态页面为主）
- **盈亏平衡**: 几乎立即
- **利润率**: 90%+

## 差异化机会

### pricepertoken.com 的不足
1. 数据更新不够及时
2. 缺少中文支持
3. 没有移动端优化
4. 缺少成本计算工具
5. 缺少 API 使用量追踪

### 我们的优势
1. **更快的数据更新** - 实时监控价格变化
2. **中英双语** - 覆盖更广的用户群体
3. **移动端优先** - 更好的移动体验
4. **成本计算器** - 增加用户价值
5. **价格提醒** - 价格变化邮件通知
6. **社区贡献** - 允许用户提交价格信息

## 风险和考量

### 风险
1. **数据准确性** - 价格变化需要及时更新
2. **版权问题** - 厂商价格数据的法律风险
3. **竞争激烈** - 已有多个类似网站
4. **流量获取** - SEO 需要时间积累

### 缓解措施
1. 建立数据验证机制
2. 添加数据来源说明
3. 寻找差异化定位
4. 多渠道流量获取
